### Ответы на контрольные вопросы

**1. Какие шаблоны активностей предоставляет Android Studio?**

- **Empty Activity** — пустая активность, универсальный шаблон.
- **Basic Activity** — включает `AppBarLayout`, `Toolbar` и `FloatingActionButton`.
- **Settings Activity** — готовый экран настроек с `PreferenceFragmentCompat`.
- **Navigation Drawer Activity** — активность с боковым выдвижным меню.

**2. Для чего используется SharedPreferences?**

Для хранения небольших объёмов данных в виде пар «ключ-значение» (настройки, состояние, простые данные пользователя).

**3. Разница между getPreferences(), getSharedPreferences() и PreferenceManager.getDefaultSharedPreferences()?**

- `getPreferences()` — настройки, доступные только в текущей активности.
- `getSharedPreferences()` — общие настройки с указанным именем.
- `getDefaultSharedPreferences()` — настройки по умолчанию для всего приложения.

**4. Как записать данные в SharedPreferences? Разница между apply() и commit()?**

```kotlin
prefs.edit().putString("key", "value").apply()
```
- `apply()` — асинхронная запись (рекомендуется).
- `commit()` — синхронная запись, возвращает результат.

**5. Как прочитать данные из SharedPreferences?**

```kotlin
val value = prefs.getString("key", "значение по умолчанию")
```

**6. Как создать экран настроек с использованием шаблона Settings Activity?**

При создании активности выбрать **Settings Activity**, элементы описываются в `res/xml/root_preferences.xml`.

**7. Как организовать переход между активностями?**

```kotlin
startActivity(Intent(this, TargetActivity::class.java))
```

**8. Что такое FloatingActionButton и где используется?**

Круглая кнопка для главного действия на экране. Присутствует в шаблонах **Basic Activity** и **Scrolling Activity**.
